# プロジェクト固有ルール (Magician YAMATO)

## Git & GitHub 運用ルール
- **コミットメッセージ**: 日本語を使用する。
- **コミットスタイル**: Conventional Commits（`feat:`, `fix:`, `chore:` などのプレフィックス）を使用する。
- **ブランチ戦略**: 本プロジェクトでは直接 `main` ブランチにコミットしてよい。（※他のプロジェクトでは原則として開発用ブランチを使用する）
- **コミットの粒度**: 作業ごとにこまめにコミットする。
- **実行の制限**: AI側で自動でコミットを実行しない。必ずコマンドやメッセージを提示し、ユーザー自身で実行するか手動の承認を経て行うこと。
